    创建缓存对象
    Parameters
    •   max {Integer} 最大缓存数
    •   buffer {Integer} 缓冲区大小
    Example
    var c=Magix.cache(5,2);//创建一个可缓存5个，且缓存区为2个的缓存对象

  • Magix. listToMap (list, key) <static>
    把列表转化成hash对象
    Parameters
    •   list {Array} 源数组
    •   key {String} 以数组中对象的哪个key的value做为hahs的key
    Returns
    • {Object}
    Example
    var map=Magix.listToMap([1,2,3,5,6]);
//=> {1:1,2:1,3:1,4:1,5:1,6:1}

var map=Magix.listToMap([{id:20},{id:30},{id:40}],'id');
//=>{20:{id:20},30:{id:30},40:{id:40}}

var map=Magix.listToMap('submit,focusin,focusout,mouseenter,mouseleave,mousewheel,change');

//=>{submit:1,focusin:1,focusout:1,mouseenter:1,mouseleave:1,mousewheel:1,change:1}

  • Magix. start (cfg) <static>
    应用初始化入口
    Parameters
    •   cfg {Object} 初始化配置参数对象
    •   cfg.appHome {String} 当前app所在的文件夹路径 http 形式的 如：http://etao.com/srp/app/
    •   cfg.debug {Boolean} 指定当前app是否是发布版本，当使用发布版本时，view的html和js应该打包成一个 view-min.js文件，否则Magix在加载view时会分开加载view.js和view.html(view.hasTemplate为true的情况下)
    •   cfg.nativeHistory {Boolean} 是否使用history state,当为true，并且浏览器支持的情况下会用history.pushState修改url，您应该确保服务器能给予支持。如果nativeHistory为false将使用hash修改url
    •   cfg.defaultView {String} 默认加载的view
    •   cfg.defaultPathname {String} 默认view对应的pathname
    •   cfg.appName {String} 应用的包名，默认app
    •   cfg.notFoundView {String} 404时加载的view
    •   cfg.routes {Object} pathname与view映射关系表
    •   cfg.appTag {String} app的资源获取时的后缀tag，增量更新时，清除缓存用
    •   cfg.iniFile {String} ini文件位置
    •   cfg.rootId {String} 根view的id
    •   cfg.ready {Function} Magix完成配置后触发
    •   cfg.extensions {Array} 需要加载的扩展
    •   cfg.execError {Function} 发布版以try catch执行一些用户重写的核心流程，当出错时，允许开发者通过该配置项进行捕获。注意：您不应该在该方法内再次抛出任何错误！
    Example
    Magix.start({
     useHistoryState:true,
     appHome:'http://etao.com/srp/app/',
     debug:true,
     appTag:'20121205',
     iniFile:'',//是否有ini配置文件
     defaultView:'app/views/layouts/default',//默认加载的view
     defaultPathname:'/home',
     routes:{
         "/home":"app/views/layouts/default"
     }
});
